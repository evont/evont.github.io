React Native 踩坑大全（持续更新）

  • React 部分

    1. 使用react-navigation 出现 Unable to resolve module 'react/lib/ReactComponentWithPureRenderMixin'的问题
      解决办法： 在package.json 中，将navigation 的配置修改为

      "React-navigation": "Git+https://github.com/react-community/react-navigation.git#7edd9a7"
      然后执行 
      rm -rf node_modules && npm instal
rm -rf android/build && rm -rf ios/build

    2. 使用真机测试时，摇晃手机无法弹出dev menu
      解决办法： 安卓手机需要开启悬浮窗权限

    3. 连接手机进行调试时，使用 adb devices 看不到已连接设备
      解决办法： 理论上是因为使用了小米手机， 进入设置 ➡️ 开发者选项 ➡️ 将启用miui 优化 选项关闭即可，如果不是，请确定手机已正确连上电脑

    4. 使用class 调用函数时找不到函数
      解决办法：

      • 在constructor 中为所有定义的方法绑定 this，如 this.XXX = this.XXX.bind(this)，缺点是 每定义一个函数都要去绑定一次，十分麻烦;
      • 在调用的时候绑定 this， 如 onPress = {this.XXX.bind(this)}，缺点是 调用同一个函数也要绑定一次this;
      • 使用箭头函数，如onPress = {()=> this.XXX()}，这种方法很简洁，也解决了this指向问题，但缺点是 ()=> 每次都会产生新函数导致不必要的性能开销;

  • Android 部分

    1. License for package XXX not accepted.
      解决办法：

      mkdir "%ANDROID_HOME%\licenses"

echo |set /p="8933bad161af4178b1185d1a37fbf41ea5269c55" > "%ANDROID_HOME%\licenses\android-sdk-license"

    2. 使用本地模拟器如天天模拟器等，无法连接的问题
      解决办法： 在启动模拟器后，根据模拟器的端口号，在命令行中执行：

      adb connect 127.0.0.1:模拟器端口

      端口号参考：

      • 天天模拟器: 6555
      • 夜神模拟器：62001
      • 蓝叠模拟器： 8700

  • iOS 部分

    1. http 请求失败的问题
      在iOS 9 或以上版本，新增了App Transport Security（简称ATS）特性，强制所有http 协议都使用https 协议进行传输，因此，如果服务器为http 的情况下，则需在info.plist 中找到 App Transport Security Settings 进行如下设置：

      • 在Exception Domains 中加入服务器域名，注意类型应为 Dictionary ；
      • 在该字典中添加 NSExceptionAllowInsecureHTTPLoads 字段，类型设置为 Boolean， 值为YES
        ;

    2. Image source size {60, 40} does not match loaded image size {30, 20}
      假设图片资源名字为xxx.png，则 xxx@2x.png 不是xxx.png 的两倍分辨率或者 xxx@3x.png 不是xxx.png 的三倍分辨率，修改图片分辨率后解决；
