建立前端错误监控系统

平日开发的时候，代码常有在开发时正常，部署到生产环境中后，因为代码的宿主环境不同而导致的兼容问题或实际细节无法复现等原因，我们常常没法轻易的查看到代码的错误原因。为了确保代码的稳定性和准确性，建立前端错误监控系统成为了一个十分有用的手段。

收集日志的方法

  1. try … catch 捕获

    在程序的入口进行try…catch 包裹，将代码同步发生的错误进行捕获，这种方法能够避免程序一跑起来就挂掉，但是如果想要全局捕获错误就显得比较无力

  2. window.onerror

window.onerror 能够有效地在全局中收集到代码运行中发生的错误，在其函数中返回 return true，错误便不会暴露到控制台中。用法如下：
