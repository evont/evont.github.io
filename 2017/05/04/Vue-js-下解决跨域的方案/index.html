使用Vue.js 或React.js 开发时跨域的解决方案

在使用Vue.js (vue-cli 创建本地项目开发) 或者React.js （create-react-app 创建本地项目开发）的时候，我们通常会遇到下面这样的跨域问题

;

那么，怎么解决这个问题呢？

Vue.js 解决方案

  1. 在项目根目录下搜索config 目录，找到index.js 文件
    ;

  2. 更改proxyTable 配置
    ProxyTable 这个参数是一个地址映射表，如图，第一个键值为代理后的路径， target 则为要请求的url，按照如下设置，则可将所有对 http://baobab.kaiyanapp.com/api/v4 的请求代理到 /api 这个 路径上，请求的时候只需要请求类似 /api／tabs/selected 的url，webpack就会自动将请求转发到 http://baobab.kaiyanapp.com/api/v4/tabs/selected 下；如果本身的请求接口存在前缀 /api ，则需在 pathRewrite 中将 '^/api' 转为 '/' 或者 ''；
    而至于跨域问题，我们可以在这个参数中加上 changeOrigin 参数，这个参数接收一个boolean 值，将其设置为true，那么本地则会代理请求，这样就解决了跨域问题；
    ;

    文档参考 Vuejs webpack Proxy配置文档
    Vue 使用的http-proxy-middleware 插件地址为 ： https://github.com/chimurai/http-proxy-middleware；

React.js 解决方案

  1. 由于create-react-app 的目标是简介的开发者体验，因此使用create-react-app 创建的项目中是看不到webpack 等的配置项的，因此我们需要先执行eject

    npm run eject
    或者 
    yarn run eject

    这个命令将所有的配置文件及package.json 依赖库解压暴露到应用所在路径，一旦执行则不可回退（真不知道这样的设计有什么意义。。。。）;

  2. 在项目根目录下搜索scripts 目录，找到start.js 文件
    ;
  3. 加入如下代码
    proxy:{
     “/api” : {
       target: “http://xxx.com/“,
       secure: false,
       changeOrigin : true
     }
    }

    这一配置与Vue.js 的ProxyTable 配置很类似，同样是将请求代理到target 所指向的url 上进行请求转发
